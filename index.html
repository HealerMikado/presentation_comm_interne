Généralité sur le web

Le web c'est quoi ?

  • Inventé par Tim Berners-Lee il y a 30 ans
  • Le web n'est qu'une application d'internet
  • Accéder à des pages à partir de leur adresse (lien hypertext)
  • Se base sur le protocole HTTP

Pour briller en société

  • 61% du traffic mondial est fait pas des bots
  • Le deep web représente 99% du web
    • deep web : pages non référencées par les moteurs de recherche
    • dark web : nécessite un logiciel particulier d'anonymisation (TOR) pour y accéder
  • Presque 2 milliards de sites existaient en octobre 2018

Comment on récupère une page web ?

L'Hypertext Transfer Protocol

  • Le web se base sur l'Hypertext Transfer Protocol (HTTP)
  • Récupérer une page se fait en deux temps
    1. La demande (HTTP request)
    2. La réponse (HTTP response)

Etape préalable : trouver l'adresse IP du site

  • 216.58.213.163 -> IP : fait pour les machines
  • Passage de l'URL à l'IP grâce une requête DNS faite aux serveurs DNS (annuaire du web)
  • Cette requête transite sur internet, donc des gens peuvent l'intercepter !

Comment on sécurise tout ça ?

En cryptant le maximum d'information

La cryptographie c'est quoi ?

Cacher un message à toutes personnes qui ne disposent pas de la clef de déchiffrement

Donc pas forcément un moyen informatique

Exemple de cryptage

  • Code césar (décalage de l'alphabet)
  • Code enigma (¯\_(ツ)_/¯)
  • Utiliser de l'argot
  • Parler dans une langue étrangère

Typologie de cryptage

  • Cryptage symétrique
  • Cryptage asymétrique

Cryptage symétrique

Une seule clef pour chiffrer et déchiffrer

Emetteur et récepteur dispose de la même clef, cryptage dans les deux sens

Comparaison avec un coffret

La clef permet de fermer (chiffrer) et ouvrir (déchiffrer) un coffret contenant un message secret

Cryptage symétrique, conclusion

Deux problèmes

  • Beaucoup trop de clefs (~n²)
  • Comment échanger la clef en toute sécurité ??

Cryptage asymétrique

  • Une clef pour chiffrer (clef publique)
  • Une pour déchiffrer (clef privée)

Cryptage à sens unique

Cryptage asymétrique, fonctionnement

  • Vous donnez à qui la demande la clef de chiffrement
  • Quelqu'un vous écrit un message et le chiffre avec cette clef
  • Vous utilisez votre clef de déchiffrement pour lire le message

Comparaison avec un coffret

Il y a une clef pour fermer le coffret (clef publique) et une clef pour l'ouvrir (clef privée)

Le chiffrement asymétrique, fonctionnement

Cryptage asymétrique, fonctionnement

La clef de chiffrement peut être largement diffusée sans risque car elle sert juste à chiffrer le message. Seul le possesseur de la clef de déchiffrement pourra le lire.

Cryptage asymétrique, conclusion

  • Chiffrement à sens unique
  • Comment discuter facilement ?

Fusionner les deux types de cryptage

Et c'est ce que fait le HTTPS

Ouf on retourne au sujet principal

HyperText transfer Protocol Secured

  • Se base sur le HTTP
  • Couche de chiffrement en plus (Transport Layer Security)

HTTPS fonctionnement

  • Demande au serveur sa clef publique
  • Génération d'une clef de session par votre ordinateur
  • Chiffrement de la clef de session par la clef publique
  • Envoi de la clef de session chiffrée
  • Déchiffrement de la clef de session par le serveur
  • Toutes les communications à partir de maintenant seront faites avec la clef de session

Comment cela fonctionne

Comparasion avec le coffret

  • On récupère la clef publique qui sert à fermer le "coffret" du serveur
  • On fabrique une clef
  • On la met dans le coffet que l'on ferme
  • On l'envoie au serveur
  • Il est le seul à pouvoir l'ouvrir avec sa clef privée
  • On utilise maintenant la clef qu'on lui a envoyé

En résumé

On utilise le chiffrement asymétrique pour pouvoir échanger en toute sécurité une clef (temporaire) de chiffrement symétrique.

Ce que le HTTPS permet

  • Chiffrer une partie de votre navigation sur le web
  • Chiffrer vos communications d'info personnelles
  • Vous assurer que le site est bien le site que vous pensez (certificat)

Ce que le HTTPS ne permet pas

  • Annonymiser totalement votre navigation (requêtes DNS, entête IP)
  • Être en parfaite sécurité que le web

Pour plus de confidentialité

  • Utiliser un proxy
  • Utiliser un VPN
  • Mais ce sont des services payants
  • Ou utiliser TOR (darknet)
